# 物业运维管理系统

## 项目概述

纯前端单页应用，用于物业运维的支出管理、合同管理、能耗管理、设备台账和日常工作记录。数据存储在浏览器 localStorage 中，支持 Excel 导入/导出。

## 技术栈

- 原生 HTML/CSS/JS（无框架）
- Bootstrap 5 样式
- SheetJS (xlsx) 用于 Excel 导入/导出
- localStorage 持久化数据

## 核心文件结构

| 文件 | 职责 |
|------|------|
| `index.html` | 主页面，包含所有导航标签页 |
| `css/style.css` | 主样式 |
| `css/responsive.css` | 响应式样式 |
| `js/app.js` | 主应用入口、路由、导航逻辑 |
| `js/storage.js` | 数据层：localStorage CRUD、自动导入、数据迁移、分类逻辑 |
| `js/modules/dashboard.js` | 仪表盘模块 |
| `js/modules/property.js` | 物业运维支出模块 |
| `js/modules/contracts.js` | 合同管理模块 |
| `data/import-data.json` | 主数据文件（初始数据源） |
| `data/物业运维支出总表.xlsx` | Excel 源文件（8个工作表） |

## 支出分类规则（核心业务逻辑）

实现位置：`js/storage.js` → `classifyPropertyExpense()` + `migrateData()`

### 分类依据

按 `item.purpose`（用途）和 `item.notes`（备注）拼接后的文本内容，用正则按**优先级从高到低**匹配关键词：

| 优先级 | 关键词（正则） | 归类为 |
|--------|---------------|--------|
| 1 | `保安\|保洁\|礼仪\|接待` | **物业费**（人工成本类） |
| 2 | `安全\|消防` | **安全费** |
| 3 | `采购\|购买\|耗材` | **物资采购** |
| 4a | 日期范围模式（如"2024年1月-2024年12月"） | **维保费** |
| 4b | `维护\|保养\|维保\|调试\|清运\|化粪池\|垃圾\|清洗\|水箱\|水质\|绿化\|养护\|虫控\|消杀` | **维保费** |
| 5 | `维修\|修理\|更换\|大修\|工程\|施工\|改造\|安装\|检验` | **维修费** |
| 6 | 以上都不匹配 | **其他** |

### 日期范围正则

```js
const DATE_RANGE_RE = /\d{4}[.\s]*年\s*\d{1,2}[.\s]*月\s*[-—~至到]\s*\d{4}[.\s]*年\s*\d{1,2}[.\s]*月/;
```

匹配形如 `2024年1月-2024年12月`、`2024.1月 至 2024.12月` 等格式。

### 旧分类名映射

```js
const CATEGORY_REMAP = {
  '物业费（保安保洁礼仪接待）': '物业费',
  '房屋和设备维修费': '维修费',
  '环境和设备维保费': '维保费',
};
```

### 费用类型列表

`js/modules/property.js` 中定义的 `EXPENSE_TYPES`：

```js
['物业费', '维保费', '维修费', '安全费', '物资采购', '其他']
```

### 数据迁移流程

`migrateData()` 在 `autoLoadData()` 中自动调用，执行两个操作：
1. 将旧分类名映射为新分类名（`CATEGORY_REMAP`）
2. 根据 `purpose` 内容重新归类（调用 `classifyPropertyExpense`）

### 维护注意事项

- 分类规则是**优先级匹配**的，第一条命中的规则即为最终分类
- 修改分类逻辑时需要同步更新 `migrateData()`，确保已有数据能正确重新归类
- 不要调整优先级顺序，除非确认业务需求变更
- 添加新关键词时注意不要与更高优先级的规则冲突
