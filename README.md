# 工作数据汇总 — 小白操作指南

## 一、如何访问网站

网站部署后你会有一个网址（如 `https://你的用户名.github.io/myweb`），用浏览器打开即可。

**推荐浏览器**：Chrome、Edge、手机自带浏览器均可。

---

## 二、首次使用：创建密码

1. 打开网站，会看到「首次使用」页面
2. 输入一个密码（至少4位），再确认一遍
3. 点击「创建密码并进入」
4. 密码用 SHA-256 加密存在浏览器里，只有你自己能进

> **重要**：这个密码是本地验证的，不是发到服务器的。请牢记密码，忘记密码无法找回（但可以通过导入备份恢复数据）。

---

## 三、首页介绍

登录后进入首页，你可以看到：

- **6 个统计卡片**：显示有效合同、合同总金额、本月能耗、本月运维支出、待复检设备、本月工作记录
- **提醒通知**：合同到期、设备待复检、能耗异常的自动提醒
- **快捷操作**：点卡片或下方按钮直接进入对应板块
- **右上角铃铛** 🔔：点击查看所有提醒通知

---

## 四、各板块操作指南

### 1. 合同目录（📋）

#### 新增合同
1. 点击「+ 新增合同」按钮
2. 填写合同信息（带 * 的必填）
   - 合同名称、编号、合作方、金额
   - 日期格式：直接用日期选择器
   - 附件：可选，支持 PDF / 图片，单文件不超过 2MB
3. 点「保存」

#### 编辑/删除
- 每条合同右侧有「编辑」和「删除」按钮
- 删除前会弹窗确认，确认后不可恢复

#### 筛选
- 用顶部的下拉框和搜索框筛选：状态、合作方、合同名称
- 点统计卡片也能快速筛选

#### 图表
- 页面底部有饼图（合同状态分布）和条形图（合作方排名）

---

### 2. 能耗用量（⚡）

#### 新增能耗记录
1. 点击「+ 新增记录」
2. 选择能耗类型（水/电/气）、月份、用量数值
3. 如有异常可勾选"异常"

#### 图表
- 月度用量趋势折线图
- 各类型累计用量条形图

---

### 3. 物业运维支出（🏗️）

#### 新增支出
1. 点击「+ 新增支出」
2. 选择支出项目类型、填写金额和日期
3. 支付方式可选

#### 大额支出
- 超过 5000 元自动标记为"大额"（阈值可在设置中修改）

---

### 4. 设备维修记录（🔧）

#### 新增维修记录
1. 点击「+ 新增记录」
2. 填写设备名称、维修日期、费用等
3. 如果维修结果选「待复检」，建议填复检日期，系统会自动提醒

---

### 5. 其他工作内容（📁）

#### 新增记录
1. 点击「+ 新增记录」
2. 填写工作名称、记录时间、分类、备注
3. 支持自定义分类

#### 管理字段
- 点击「📐 管理字段」可以添加/删除/修改表单字段

---

### 6. 数据汇总（📊）

#### 查看总报表
- 一页看完所有板块的核心数据
- 支持按时间周期筛选
- 点击「📤 导出总报表」下载 text 格式报表

#### 导出 Excel
- 点击「📥 导出Excel」
- 每个板块一个 Sheet，可以直接在 Excel 中打开编辑

## 五、数据备份（非常重要！）

### 为什么需要备份？

数据存在浏览器里。以下情况会丢数据：
- 清除浏览器数据
- 重装系统
- 换电脑

### 如何备份？

**方法一：JSON 文件备份（本地）**
1. 进入「设置」页面
2. 点击「📤 导出全部数据 (JSON)」
3. 浏览器会下载一个 `.json` 文件，把它存到安全的地方（如 D 盘、U 盘）

**恢复方法**：
1. 进入「设置」页面
2. 点击「📥 导入数据恢复」
3. 选择之前的 `.json` 文件
4. 确认后数据恢复，页面自动刷新

**方法二：GitHub 云端同步（推荐）**
1. 创建一个 GitHub 私有仓库
2. 生成 Personal Access Token（需要 `repo` 权限）
3. 在「设置 → GitHub 云端同步」填入 Token 和仓库名
4. 点击「保存配置」
5. 点击「📤 推送到云端」上传数据
6. 换电脑或重装后，点击「📥 从云端拉取」恢复

**建议**：每月备份一次，重要操作前先备份。

## 七、常见问题

**Q: 数据存在哪里？**
A: 存在浏览器的 localStorage 里，不上传到任何服务器。所以你的数据完全私密，但也要做好备份。

**Q: 密码忘了怎么办？**
A: 密码无法找回。但你之前备份的 JSON 文件里不包含密码，可以通过"清空所有数据"重置网站，然后用导入恢复数据重新设置密码。

**Q: 附件能传多大的？**
A: 限制 2MB。如果文件太大，建议压缩或用截图代替。

**Q: 手机能用吗？**
A: 可以，界面会自动适配手机屏幕。

**Q: 需要联网吗？**
A: 日常录入和查看不需要。只有以下操作需要网络：GitHub 云端同步、首次加载 CDN 资源（Chart.js 等）。

**Q: CDN 加载失败怎么办？**
A: 检查网络连接。CDN 资源只首次加载，浏览器会缓存。
